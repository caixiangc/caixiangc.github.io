在实际开发的时候，就可知，实际真实设计稿尺寸 = dpr * 理想视口的尺寸(就是我们依据psd量出的宽)，也就是说只要我们获取到不同设备的dpr，我们就能通过一份代码写出n 份 不同设备的响应式

4.2 理解

设备像素比(dpr) 是指在移动开发中1个css像素占用多少设备像素，如2代表1个css像素用2x2个设备像素来绘制。

设备像素比(dpr)，公式为 1px = (dpr)^2 * 1dp ，可以理解为1px由多少个设备像素组成；

所以//凡是设计像素的 都要 乘以 设备像素比(像 宽，高，fontsize，图片宽高也都是要的，。。都是)

redis高迸发分布式锁

(redis是单线程服务的，就是里边的程序一次只能由一个进程执行，支持1s 几十万的迸发)
1.只能锁一个线程
2.value-》productId ，key=》time
3.解锁的方式两种：一是程序执行完自动解锁，二是程序超时被取代
4.低迸发，如果一个线程出现异常没执行完，但是还在超时范围以内，那是不会被新的value取代的（重要）
5.在高迸发的情况中，都是会主线 相同的 key 和value 但是他们的线程不同，所以他们还是有先后顺序的。（重要）
6.在高压测试的时候，比如1s 500 个请求，但是实际一秒 只执行了 10+的请求，就是因为分布式锁，一些进程没有得到锁，所以就没有执行（这种事防止超库存，就是超卖）

用法一般是在你需要严格控制线程的业务代码上 上锁（如秒杀业务中的扣库存）

Maven 打包问题

项目完成后

如果启动报错：
org.springframework.beans.factory.BeanCreationException: Error creating bean with name ‘entityManagerFactory’ defined in class path resource [org/springframework/boot/autoconfigure/orm/jpa/HibernateJpaAutoConfiguration.class]: Invocation of init method failed; nested exception is java.lang.NoClassDefFoundError: javax/xml/bind/JAXBException
如果报错这个，那么报错信息在后面：
nested exception is java.lang.NoClassDefFoundError: javax/xml/bind/JAXBException

百度后可知
是因为java 版本的问题。
要导入


​ javax.xml.bind
​ jaxb-api
​ 2.2.12

这个包，然后重新打包就行了

执行
1.maven打包(跳过测试打包)：
mvn clean package -Dmaven.test.skip=true
（打包后的jar 文件 就是我们需要的jar文件）
2.执行包:
nohup java -jar aaa.jar& //后面要加 & 就是关闭命令行它也运行
3.要关闭正在运行的jar包 ，直接杀死进程就行了
4.sudo lsof -i :9000 （查看端口9000的pid），也可以通过
ps -ef | grep sell.jar 来查看启动sell.jar 的pid
5.sudo kill -9 pid

也可以写一个shell 脚本 通过bash 来启动
这个很重要，当我们要写其他脚本的时候就可以这样 .sh 来新建一个脚本

//编辑脚本(创建脚本)
atom start.sh

// 启动脚本

bash start.sh

//查看是否启动(看jar 包或者端口)

sudo lsof -i :8085
