Nginx 的一些用法 和注意事项

查看Nginx的版本号：nginx -V

启动Nginx：start nginx

快速停止或关闭Nginx：nginx -s stop

正常停止或关闭Nginx：nginx -s quit

配置文件修改重装载命令：nginx -s reload

查看windows任务管理器下Nginx的进程命令：tasklist /fi “imagename eq nginx.exe”

如果在本地需要配置虚拟域名 那么需要修改 系统host 配置文件，在里面添加10.31.172.22 image.test.com

就是如果访问 image.test.com那么转发到这个ip(10.31.172.22)地址。这样我们可以配置图片的服务器地址，或者后台，或者前台的服务器地址，实现完全无关的分离

nginx.exe -t是验证配置文件是否正确的

autoindex on;自动索引（会对那个服务器地址自动安排索引树形结构排出来），一般是需要关闭的，当我们访问image.mymall.com的时候 这里配置的是off，如果我们访问image.mymall.com的时候会报错403（无权限），但是对于里面的资源是不受影响的可以正常访问

全局变量和静态变量

1.私有方法优先级大于全局（如果私有方法变量未定义，那么用全局的）

2.在私有方法中定义了一个与全局变量同名的变量是，是改变和使用私有方法，改变值无法保存到全局

存储类型

​ 1.static 静态 static int a=2; //如果在私有方法中 定义了static 变量，那么这个方法执行完变量不会被回收，下次调用这个方法还能继续使用改变后的变量

​ 2.auto 动态 （默认）int a=2 //如果不定义static 变量 那么私有方法运行完里边的变量，都会被系统回收

int b ：分配存储空间

b=3 ：在原有的空间 上定义或者改变值

int b=3 ：在分配存储空间的时候就赋上值

static int b=3 ：在分配存储空间的时候就赋上值静态方法

JS事件冒泡和事件捕捉

在学校，听老师讲解事件冒泡和事件捕获机制的时候跟听天书一样，只依稀记得IE使用的是事件冒泡，其他浏览器则是事件捕获。当时的我，把它当成IE浏览器兼容问题，所以没有深究（IE8以下版本的浏览器已基本退出市场）。工作至今，虽然多次遇到该类问题，但均未深究，始终一知半解，遇到了全TM靠猜（选A不行就选B呗）。今天闲来无事自己做了个demo,算是把这个问题彻底搞明白了。

先上结论：他们是描述事件触发时序问题的术语。事件捕获指的是从document到触发事件的那个节点，即自上而下的去触发事件。相反的，事件冒泡是自下而上的去触发事件。绑定事件方法的第三个参数，就是控制事件触发顺序是否为事件捕获。true,事件捕获；false,事件冒泡。默认false,即事件冒泡。Jquery的e.stopPropagation会阻止冒泡，意思就是到我为止，我的爹和祖宗的事件就不要触发了。
