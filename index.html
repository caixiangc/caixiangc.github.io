redis高迸发分布式锁

(redis是单线程服务的，就是里边的程序一次只能由一个进程执行，支持1s 几十万的迸发)
1.只能锁一个线程
2.value-》productId ，key=》time
3.解锁的方式两种：一是程序执行完自动解锁，二是程序超时被取代
4.低迸发，如果一个线程出现异常没执行完，但是还在超时范围以内，那是不会被新的value取代的（重要）
5.在高迸发的情况中，都是会主线 相同的 key 和value 但是他们的线程不同，所以他们还是有先后顺序的。（重要）
6.在高压测试的时候，比如1s 500 个请求，但是实际一秒 只执行了 10+的请求，就是因为分布式锁，一些进程没有得到锁，所以就没有执行（这种事防止超库存，就是超卖）

用法一般是在你需要严格控制线程的业务代码上 上锁（如秒杀业务中的扣库存）

Maven 打包问题

项目完成后

如果启动报错：
org.springframework.beans.factory.BeanCreationException: Error creating bean with name ‘entityManagerFactory’ defined in class path resource [org/springframework/boot/autoconfigure/orm/jpa/HibernateJpaAutoConfiguration.class]: Invocation of init method failed; nested exception is java.lang.NoClassDefFoundError: javax/xml/bind/JAXBException
如果报错这个，那么报错信息在后面：
nested exception is java.lang.NoClassDefFoundError: javax/xml/bind/JAXBException

百度后可知
是因为java 版本的问题。
要导入


​ javax.xml.bind
​ jaxb-api
​ 2.2.12

这个包，然后重新打包就行了

执行
1.maven打包(跳过测试打包)：
mvn clean package -Dmaven.test.skip=true
（打包后的jar 文件 就是我们需要的jar文件）
2.执行包:
nohup java -jar aaa.jar& //后面要加 & 就是关闭命令行它也运行
3.要关闭正在运行的jar包 ，直接杀死进程就行了
4.sudo lsof -i :9000 （查看端口9000的pid），也可以通过
ps -ef | grep sell.jar 来查看启动sell.jar 的pid
5.sudo kill -9 pid

也可以写一个shell 脚本 通过bash 来启动
这个很重要，当我们要写其他脚本的时候就可以这样 .sh 来新建一个脚本

//编辑脚本(创建脚本)
atom start.sh

// 启动脚本

bash start.sh

//查看是否启动(看jar 包或者端口)

sudo lsof -i :8085

本地代码git 到github

1.在github上创建好存储空间repository

2.touch .gitignore

  1. git init// 会生成一个 .ssh 文件，但是不可见
  1. git status//可以看到这个项目中有那些 项目发生变化，当作是验证就可以了
  1. git add .//把所以文件添加到git 的本地仓库（这里是准备上传的）
  1. git commit -m 'first commit init project'//-m 就是后面添加注释
  1. git remote add origin git@........
  1. git brance//查看分支
  1. git push -u origin master//现在正式连接远程仓库上传

管理分支

​

  1. git branch //查看有几个分支
  2. git checkout ‘分支名’ //你想要转换到的分支名
  3. git add . //如果是不同版本 ，那就add 所有
  4. git commit -m “” //一定要commit 否者不能push
  5. git push -u origin ‘分支名’ //你要添加的分支名

这个是用branch 提交不同版本的

//git 在切换分支的时候，要预先创建readme文件
git pull —rebase origin master

然后具体步骤：
1.git add .
2.git remote add origin https://github.com/你的用户名/github项目名.git
3.git push -u origin master

只有在那个branch 下才能push 到远程 对应的那个branch

*要小心切换分支，每次切换 相当于换个文件夹**
git push origin second2_6:master -f
//强制 把 second2_6分支代码push到远程master分支

本地做好修改再去push到远端，所以我推荐如下操作
git checkout master

git reset –hard develop
//先将本地的master分支重置成develop,要先切换到master分支上

git push origin master –force //再推送到远程仓库

Nginx 的一些用法 和注意事项

查看Nginx的版本号：nginx -V

启动Nginx：start nginx

快速停止或关闭Nginx：nginx -s stop

正常停止或关闭Nginx：nginx -s quit

配置文件修改重装载命令：nginx -s reload

查看windows任务管理器下Nginx的进程命令：tasklist /fi “imagename eq nginx.exe”

如果在本地需要配置虚拟域名 那么需要修改 系统host 配置文件，在里面添加10.31.172.22 image.test.com

就是如果访问 image.test.com那么转发到这个ip(10.31.172.22)地址。这样我们可以配置图片的服务器地址，或者后台，或者前台的服务器地址，实现完全无关的分离

nginx.exe -t是验证配置文件是否正确的

autoindex on;自动索引（会对那个服务器地址自动安排索引树形结构排出来），一般是需要关闭的，当我们访问image.mymall.com的时候 这里配置的是off，如果我们访问image.mymall.com的时候会报错403（无权限），但是对于里面的资源是不受影响的可以正常访问

nginx 中的server_name sell.com (就是你的域名)
